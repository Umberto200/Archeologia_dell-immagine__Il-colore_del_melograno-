Metadati per le scene del film

  • Tipo di file: RDF/XML
  • Descrizione: Ogni fotogramma o scena è descritto con metadati strutturati come titolo, creatore, soggetto, descrizione testuale, timestamp, editore e diritti, utilizzando il vocabolario Dublin Core.
  • Esempio: 
    <dc:title>f1.2</dc:title>
<dc:creator>Sergej Paradžanov</dc:creator>
<dc:description>Sequenza iniziale di immagini evocative...</dc:description>
  • Tecnologie/Standard utilizzati:
    • RDF (Resource Description Framework): Framework per descrivere risorse sul web.
    • XML (eXtensible Markup Language): Formato di serializzazione per i dati strutturati.
    • Dublin Core (dc): Vocabolario standard per i metadati.

Metadati per le poesie

  • Tipo di file: XML-TEI
  • Descrizione: Le poesie sono codificate seguendo gli standard TEI, con elementi strutturali (versi, strofe, titoli) e metadati (autore, data, lingua).
  • Esempio: 
    <teiHeader>
  <fileDesc>
    <titleStmt><title>Poema XYZ</title></titleStmt>
    <publicationStmt><publisher>Editore</publisher></publicationStmt>
  </fileDesc>
</teiHeader>
  • Tecnologie/Standard utilizzati:
    • TEI (Text Encoding Initiative): Standard per la codifica di testi digitali.
    • XML: Formato di base per la strutturazione dei dati.

Metadati per i simboli

  • Tipo di file: Turtle (.ttl)
  • Descrizione: Oggetti simbolici (es. un pugnale) sono descritti come risorse fisiche con attributi visivi, posizione nella scena e relazioni semantiche, utilizzando RDF in formato Turtle.
  • Esempio: 
    :Pugnale a dc:PhysicalResource ;
    dc:title "Pugnale" ;
    dc:description "Rosso, grigio, nero" ;
    dc:coverage "in primo piano" .
  • Tecnologie/Standard utilizzati:
    • Turtle (Terse RDF Triple Language): Formato compatto per RDF.
    • RDF: Framework per dati collegati semantici.
    • Dublin Core: Esteso per descrizioni specifiche.
